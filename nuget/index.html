Что такое NuGet?

Набор программ, которые интегрируются в Visual Studio для упрощения использования сторонних библиотек в собственных проектах.

  • Visual Studio 2010 - расширение, через Extension Manager
  • Visual Studio 2013 - встроен "из коробки"

Что он умеет?

  1. Искать нужную библиотеку
  2. Загружать пакет, который содержит нужную библиотеку
  3. Проверять целостность пакета
  4. Распаковывать пакет в нужное место нашего проекта
  5. Добавлять к нашему проекту связи с библиотеками из состава пакета
  6. Добавлять в файлы конфигурации нашего проекта нужные параметры
  7. Следить за обновлениями библиотек и доставлять их в наш проект

Можно без GUI?

Package Manager Console - использует Powershell для текстового управления NuGet

  • Get-Package -ListAvailable -Filter строка поиска для поиска пакета в сети
  • Install-Package имя пакета для установки пакета
  • Update-Package имя пакета для обновления пакета

... и многое другое

Откуда берутся пакеты?

  1. Официальная галерея NuGet
  2. Неофициальные источники, заданные через URL
  3. Локальные или сетевые папки

    Источники, из которых можно забирать пакеты, настраиваются

Куда попадают пакеты после установки?

  • В каталог packages внутри каталога решения
  • Никуда. Серьезно.

Как делаются пакеты?

  1. Создать одну или несколько библиотек, которые должны быть включены в пакет
  2. Сгенерировать манифест пакета
  3. Обновить метаданные сборок, которые добавляются в пакет
  4. Создать пакет при помощи nuget.exe

Манифест пакета

			<?xml version="1.0"?>
			<package xmlns="...">
  				<metadata>
  					<id>$id$</id>
  					<version>$version$</version>
  					<authors>$authors$</authors>
  			<!-- ... и еще несколько служебных полей-->

Способы создания пакета

  • Вручную при помощи xml-файлов, nuget.exe и текстового редактора
  • Автоматически при помощи NuGet Package Explorer и магии :-)

Как NuGet сделает добро для нас?

  • Упростит интеграцию большого количества внешних компонент в сложный проект (например, rotapost.ru)
  • Упростит повторное использование одинаковых библиотек в разных проектах в составе одного решения
  • При помощи TeamCity позволит оформить собственный источник пакетов, в которых будут самые часто используемые нами самописные библиотеки
